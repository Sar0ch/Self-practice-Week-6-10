First-Class Function หมายถึง ฟังก์ชันในภาษานั้น ถูกปฏิบัติเหมือนค่าทั่วไป (value)
คือสามารถ “เก็บ”, “ส่ง”, และ “ใช้” ได้เหมือนกับตัวแปรตัวเลขหรือ string ทั่วไป


Functional Prpgraming
higer order function คือ fucntion ที่ return fucntion ออกไป หรือ รับ function เป็น parameter 
filter , map ถือเป็น higher เพราะ รับ callback fucntion
ข้อแนะนำในการออกแบบ function 
- ไม่ควรเปลี่ยน original data ไม่ควรเปลี่ยนโครงสร้างข้อมูลภายใน map/filter มันได้ new array

higer order มันเหมือนเป็น container fucntion ที่มาประกบมันที่ทำให้มันทำงานได้มากขึ้น 
การที่เราออกแบบให้เป็น higher order คือ ทำให้มันทำงานได้มาก หลากหลายขึ้น ฟิลใส่ logic เข้าไปใน function 

Higher-Order Function จะเกิดขึ้นได้ก็ต่อเมื่อ ฟังก์ชันเป็น First-Class

คุณลักษณะสำคัญของการเขียนโปรแกรมเชิงฟังก์ชัน (Functional Programming)❤️
1.Single Responsibility - ออกแบบฟังก์ชั่นให้เป็น unit ที่เล็กที่สุด มี 1 ความรับผิดชอบ 
2.Declarative - เขียนแบบบอก “สิ่งที่ต้องการ” มากกว่าบอก “ขั้นตอน”
3.Immutability - ไม่เปลี่ยนแปลงข้อมูล โครงสร้างของ original สร้างขึ้นมาใหม่แทน

ถ้าอยากออกแบบฟังก์ชั่นให้ไป compose กับ ฟังก์ชั่นอื่น ต้องออกแบบให้เป็น Pure function 
คือ input เดียวกันต้องได้ output เดียวกันเสมอ 
หลักการคือห้ามแตะต้องตัวแปรนอกสโคปฟังก์ชั่นของตัวเอง 
ตัวแปรภายในฟังก์ชั่นมีกี่อัน ตัวแปรภายในังก์ชั่นจะถูกทำลายเมื่อสิ้นสุดลง
1.parameter 
2.ตัวแปรที่สร้างขึ้นใหม่ในสโคปนั้น หรือก็คือใน {} 
อยากให้เป็น Pure ต้องชัว output
